# 评估标准定义

## 评估维度与权重

| 维度 | 权重 | 说明 |
|------|------|------|
| 架构设计 | 20% | 整体架构的合理性、可维护性 |
| 核心能力 | 25% | 代码生成、理解、编辑等核心功能 |
| 技术实现 | 20% | LLM集成、Prompt工程、RAG等技术细节 |
| 交互体验 | 15% | CLI/IDE交互、权限控制、确认机制 |
| 安全机制 | 10% | 沙箱隔离、权限模型、数据隐私 |
| 可扩展性 | 10% | 插件系统、MCP支持、工具注册 |

## 评分标准

### 5分制

| 分数 | 含义 |
|------|------|
| 5 | 业界领先，创新性强 |
| 4 | 优秀，有明显优势 |
| 3 | 良好，符合行业平均水平 |
| 2 | 一般，有改进空间 |
| 1 | 较弱，需要较大提升 |

## 评估原则

1. **客观性**：基于事实和数据，避免主观臆断
2. **一致性**：所有 Agent 使用相同的标准评估
3. **可验证性**：每个评估结论都应有源码或文档支撑
4. **时效性**：基于最新版本进行评估
