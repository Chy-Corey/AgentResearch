# 测试方法

## 测试场景

### 场景一：代码生成
- 简单函数生成
- 复杂模块生成
- 多文件项目生成

### 场景二：代码理解
- 代码解释
- Bug 定位
- 架构分析

### 场景三：代码编辑
- 单文件修改
- 跨文件重构
- 依赖更新

### 场景四：工具使用
- 文件系统操作
- Git 操作
- 终端命令执行

## 测试指标

| 指标 | 说明 | 测量方式 |
|------|------|----------|
| 准确性 | 生成代码的正确率 | 单元测试通过率 |
| 完整性 | 功能覆盖程度 | 需求满足率 |
| 效率 | 完成任务所需时间 | 端到端时间 |
| 安全性 | 安全机制的有效性 | 安全测试通过率 |
