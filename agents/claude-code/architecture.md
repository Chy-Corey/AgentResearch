# Claude Code — 架构设计分析

本文档从整体架构、模块划分、数据流、状态管理、通信机制五个维度分析 Claude Code 的架构设计。

## 整体架构

### 架构模式

### 架构特点

| 特点 | 说明 |
|------|------|
| **分层清晰** | 用户交互 → 命令解析 → Agent 核心 → 服务 → 基础设施 |
| **模块化** | 每个功能独立模块，可单独开发和测试 |
| **可扩展** | MCP 协议支持第三方工具扩展 |
| **响应式** | 状态变化自动触发 UI 更新 |
| **流式处理** | LLM 响应和工具执行都是流式的 |

## 状态管理

### 状态存储架构

Claude Code 使用**自定义 Store**（类似 Redux）管理状态：

```typescript
// state/store.ts - 核心 Store 实现
type Store<T> = {
  getState: () => T;                    // 获取当前状态
  setState: (updater: (prev: T) => T) => void;  // 更新状态
  subscribe: (listener: () => void) => () => void;  // 订阅变化
};

function createStore<T>(initialState: T, onChange?: OnChange<T>): Store<T> {
  let state = initialState;
  const listeners = new Set<() => void>();

  return {
    getState: () => state,
    setState: (updater) => {
      const prev = state;
      const next = updater(prev);
      if (Object.is(next, prev)) return;  // 状态没变，不通知
      state = next;
      onChange?.({ newState: next, oldState: prev });
      for (const listener of listeners) listener();  // 通知所有订阅者
    },
    subscribe: (listener) => {
      listeners.add(listener);
      return () => listeners.delete(listener);
    },
  };
}
```

### 全局状态结构

```typescript
// state/AppStateStore.ts - 全局状态定义
type AppState = {
  // 用户设置
  settings: SettingsJson;
  verbose: boolean;
  
  // 模型配置
  mainLoopModel: ModelSetting;
  
  // 对话消息
  messages: Message[];  // 实际存在 state 中，通过 ref 访问
  
  // 工具和命令
  tools: Tools;
  commands: Command[];
  
  // 权限
  toolPermissionContext: ToolPermissionContext;
  permissionMode: PermissionMode;
  
  // MCP 服务器
  mcpServers: MCPServerConnection[];
  
  // 插件
  plugins: LoadedPlugin[];
  
  // 任务
  tasks: TaskState;
  
  // Bridge（远程控制）
  replBridgeEnabled: boolean;
  replBridgeConnected: boolean;
  
  // UI 状态
  expandedView: 'none' | 'tasks' | 'teammates';
  footerSelection: FooterItem | null;
};
```

### 状态更新流程

```
用户操作 / LLM 响应
  ↓
触发状态更新
  ↓
appState.setState(prev => ({
  ...prev,
  messages: [...prev.messages, newMessage]
}))
  ↓
Store 检查状态是否变化
  ↓
如果变化，通知所有订阅者
  ↓
UI 组件重新渲染（Ink/React）
```

### 状态订阅机制

```typescript
// UI 组件订阅状态变化
function MessageList() {
  const [messages, setMessages] = useState([]);
  
  useEffect(() => {
    // 订阅状态变化
    const unsubscribe = appState.subscribe(() => {
      setMessages(appState.getState().messages);
    });
    
    return unsubscribe;  // 组件卸载时取消订阅
  }, []);
  
  return (
    <Box>
      {messages.map(msg => <Message key={msg.id} message={msg} />)}
    </Box>
  );
}
```

## 通信机制

### 1. 函数调用（同进程）

最直接的通信方式，用于模块间调用：

```typescript
// query.ts 调用 tools.ts
const tools = getTools();

// tools.ts 调用 Tool.execute()
const result = await tool.execute(input, context);

// main.tsx 调用 query.ts
for await (const event of query(params)) {
  handleEvent(event);
}
```

### 2. 状态订阅（响应式更新）

用于 UI 自动更新：

```typescript
// 状态变化 → UI 更新
appState.setState(prev => ({ ...prev, messages: [...prev.messages, msg] }));
  ↓
所有订阅者被通知
  ↓
UI 组件重新渲染
```

### 3. 事件流（AsyncGenerator）

用于流式处理 LLM 响应：

```typescript
// query.ts 生成事件流
async function* query(params): AsyncGenerator<StreamEvent> {
  while (true) {
    const stream = await callModel(messages);
    for await (const event of stream) {
      yield event;  // 产出事件
    }
  }
}

// UI 消费事件流
for await (const event of query(params)) {
  if (event.type === 'text') {
    displayText(event.text);
  }
}
```

### 4. MCP 协议（进程间通信）

用于与外部工具通信：

```
Claude Code（主进程）
  ↓ MCP 协议（JSON-RPC over stdio/WebSocket）
MCP 服务器（独立进程）
  ↓
外部工具（Jira、Slack、Chrome 等）
```

### 5. Bridge 协议（远程通信）

用于远程控制和跨设备协作：

```
本地 Claude Code
  ↓ WebSocket
claude.ai 云端
  ↓ WebSocket
远程设备（手机/其他电脑）
```
